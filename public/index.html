Documentation

Parrot is a an HTTP API to consume limited information from multiple social platforms. The design is intentionally simple and transparent. There's really only one endpoint.

GET https://parrot.crz.li/<platform>/<username>.<format>

The response comprises that user's latest activity on that platform in the desired format. For instance, if we were to make a request to.

GET https://parrot.crz.li/twitter/geoffkeighley.html

Parrot would hit the Twitter API, authenticates, get an access token, fetch the latest status from @geoffkeighley and return it in format appropriate to be embedded in HTML.

HTTP/1.1 200 OK
Content-Type: text/html; charset=utf-8

Exactly 3 months until <a href="https://twitter.com/thegameawards">@thegameawards</a> - can’t believe the show that’s coming together this year!

Supported platforms

Platform Identifier Spec
Twitter twitter The content of the latest twit.
Instagram instagram The URL of the latest 5 photos. Requires authentication.

Some platforms may require additional authentication to work. Such is the case of Instagram which requires the owner of the data to give explicit permission to have their data read by Parrot.

The process is as simple as it can be;

  1. A request to an unauthenticated profile will get a redirection to the platform's authentication screen.
  2. From there you will be redirected back to Parrot and the process will be completed.

Subsequent requests will now fetch and return the expected results. Please note that although we momentarily cache data to avoid service interruption this authorization will eventually expire and it's the user's responsibility to repeat the authentication procedure.

Supported formats

Format Identifier Spec
HTML html Image URLs as <img> tags; mentions and other URLs as anchors.
JSON json Structured JSON.
Text txt Plain text with no transformations. Multiple entries separated by new lines.

FAQ

What is Parrot?

Parrot is a service that fetches your latest activity on social media to have it embedded elsewhere, like on your blog for instance.

How much does it cost?

Currently Parrot is free of charge. As the product matures we might have to find ways to make it sustainable.

What platforms does it work with?

Right now it’s just Twitter. We’re actively working on integrating more platforms such as Instagram.

What are the limitations?

There are some. Not only because of limitations of the platform itself, but also because of how Parrot was designed. First of all it only works with public data. The content must be cached for a small amount of time (minutes usually, an hour at most) so it’s not in real-time. Also we don’t store historical data, so once the cache is refreshed the previous content is automatically discarded.
